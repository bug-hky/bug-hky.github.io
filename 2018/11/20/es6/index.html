es6的一些优雅写法

发表于 2018-11-20 | 阅读次数:

    js进化成es6/7/8之后，有一些优雅的写法可以避免增加自己代码后期维护的时间成本，实际上es5和jquery制霸的时候有些的写法太过糟糕。其实也有办法，会有些麻烦，所以有些重复的代码由于项目周期等不可抗力，并没有时间来优化，久而久之形成习惯对开发者是一件挺可怕的事，当然对于一些对产品质量没有底线的项目甲方来说这些无关紧要，但这对于迭代项目简直是灾难，这点也是参与迭代项目后体会到的。还有es6本身新增的一些拓展也十分的强大，至少我在项目中运用到的可能连它十分之一都没有用到。并且自己的写法过于冗长，可读性基本没有，想实现业务有很多种办法，毕竟现在有了mvvm，更是随心所欲，es6其实有很多方法法可以优化我们的项目，总结一部分，以便于梳理以后写东西的思路。

数组去重

去重是非常常见的操作了
Set本身是一个构造函数，和数组长得很像，但是没有重复值，所以可以利用这一特性做一些事，…是拓展运算符，可以把部分数据结构转化为数组(Map & Set)，原理是通过调用Symbol.iterator，这和array.from方法有本质的区别。

基础数组

const arr = [1, 2, 2, 3, 4, 5];
[...newSet(arr)]
=> [1, 2, 3, 4, 5]

复杂条件判断

if/else/switch在一般的业务场景下其实已经够用了，但是在碰到多元的条件判断时，由于时间紧，往往便选择了if/else嵌套的方式来实现，这种方式实际上既增加了代码量，又多写了重复的判断条件以及同样的操作。

场景

/*
admin: 多级销售 && 多级用户 && 游客
>> userType: custom => 客户 || sales => 销售
>> level: 1 => level1 || 2 => level2 || 3 => level3 || 4 => level4 || 5 => level5
*/
const funcs = new Map([
  ['custom_1', ()=>{}],
  ['custom_2', ()=>{}],
  ['custom_3', ()=>{}],
  ['custom_4', ()=>{}],
  ['custom_5', ()=>{}],
  ['sales_1', ()=>{}],
  ['sales_2', ()=>{}],
  ['sales_3', ()=>{}],
  ['sales_4', ()=>{}],
  ['sales_5', ()=>{}],
  ['tourist', ()=>{}],
])

const doSth = (userType, level)=>{
  let action = funcs.get(`${userType}_${level}`) || funcs.get('tourist')
  action();
}
